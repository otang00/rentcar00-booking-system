# RENTCAR00 NOTIFICATION PREP CURRENT

## 문서 상태
- 상태: prep current
- 목적: 예약 원장 생성 후 알림 구현 전 선행조건과 즉시 구현 가능 범위를 잠근다.

## 1. 알림 채널 분리
### 운영자 알림
- 운영 이메일: 즉시 구현 가능
- 관리자 알림톡: 이번 구현 범위 포함

### 고객 알림
- 고객 알림톡: 최종 필요
- 단, 서비스 신청/채널 준비/템플릿 승인 선행 필요
- 따라서 이번 단계에서는 구현 blocker 로 둔다.

## 2. 즉시 구현 가능 범위
1. 원장 생성 직후 운영 이메일 발송
2. 원장 생성 직후 관리자 알림톡 발송
3. 두 채널 모두 성공/실패 이벤트 적재
4. 실패해도 예약 생성 자체는 롤백하지 않음

## 3. 고객 알림톡 선행조건
1. 발송 서비스 신청 상태 확인
2. 채널/발신 프로필 준비 확인
3. 템플릿 승인 여부 확인
4. 고객 안내 문구 확정
5. 실패/재시도 운영 기준 확정

## 4. 구현 전 체크포인트
- 운영 이메일 수신처가 현재와 동일한지 확인
- 관리자 알림톡 수신 대상을 누구로 잠글지 결정
- Solapi에서 문자/알림톡 API 경로를 어떻게 나눌지 결정
- 고객 알림톡 준비 전에는 코드에 TODO와 비활성 기준을 명시

## 5. 이번 단계 완료 기준
- 운영 이메일/관리자 알림톡은 구현 대상으로 잠김
- 고객 알림톡은 blocker 와 선행조건이 문서화됨
- 실행 current 문서와 모순이 없음
