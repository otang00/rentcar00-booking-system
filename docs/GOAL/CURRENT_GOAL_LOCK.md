# CURRENT GOAL LOCK - 차량그룹 가격정책 정리 + 차량 운영 노출 관리

## Primary Goal
- 현재 가장 중요한 목표: **차량그룹 기준으로 가격정책을 다시 정리하고, 관리자 메뉴에서 차량 1대별 노출/대여 가능 상태를 직접 제어할 수 있게 한다.**

## Why This Matters
- IMS 기준으로 따라온 가격정책이 실제 운영 기준과 어긋나 있어, 같은 차량그룹인데 정책이 불필요하게 나뉘거나 반대로 같이 묶이면 안 되는 항목이 섞여 있다.
- 운영자가 차량 1대별로 홈페이지 노출 여부와 15일 이상 장기 대여 허용 여부를 직접 제어할 수 있어야 검색 결과와 운영 판단이 안정된다.
- 가격정책 정리와 차량 노출 기준을 분리해 두어야 IMS sync와 홈페이지 운영 기준이 서로 덜 꼬인다.

## Current Context
- 가격 계산은 현재 `source_group_id` 기준 가격정책 연결을 통해 이뤄진다.
- 같은 차량그룹에 같은 가격정책을 연결하는 방향은 현재 코드 구조상 가능하며, 주 변경 대상은 코드보다 `price_policy_groups` 연결 구조다.
- 검색 가격은 차량그룹 기준으로 보이지만, 예약 직전 availability 최종 검사는 `source_car_id` 1대 기준 IMS/홈페이지 예약 중복을 확인한다.
- 최근 완료된 작업:
  - 최종 노출/예약확정 가격 라운딩 정리 완료
  - 관리자 예약 변경 UI MVP 완료
- 다음 작업은 두 갈래다.
  1. 차량그룹별 가격정책 정리
  2. 관리자 차량운영관리 메뉴 신설

## Success Criteria
- 차량그룹별 가격정책 연결표가 정리되어 운영자가 어떤 그룹이 어떤 정책을 쓰는지 명확히 볼 수 있다.
- 불필요하게 분리된 가격정책은 같은 차량그룹 기준으로 정리할 수 있다.
- 관리자 메뉴에서 차량 1대별로 아래 두 값을 수정할 수 있다.
  - 대여가능
  - 15+일 대여
- 홈페이지 검색은 아래 기준으로만 노출된다.
  - 차량 기본 활성 조건 충족
  - 운영자 `대여가능` 체크 차량만 노출
  - 15일 이상 검색이면 `15+일 대여` 체크 차량만 노출
- 예약 직전 최종 중복 검사는 계속 차량 1대 기준으로 유지된다.

## Active Constraints
- 목표 문서는 실행 승인 문서가 아니다.
- 코드 수정, DB migration, 배포, 외부 반영은 별도 승인 전 금지한다.
- 가격정책 연결 정리는 코드보다 데이터/관리 구조 영향이 크므로, 현재 연결표 확인과 반영 단계를 분리한다.
- `.env`, secret, 운영 설정, 배포 설정, 운영 DB 직접 반영은 protected target이다.
- IMS sync 원본 필드를 운영 편의로 덮어쓰지 않는다.
- 차량 운영 노출 제어는 IMS 필드가 아니라 홈페이지 전용 local 기준으로 분리하는 것을 우선 검토한다.

## Next Actions
1. 현재 차량그룹 ↔ 가격정책 연결표를 운영 검토용으로 정리한다.
2. 그룹명 / 차량번호 / 가격정책만 수정하는 검토용 HTML을 안정적으로 정리한다.
3. 차량 1대별 운영 상태를 위한 관리자 메뉴 구조를 설계한다.
4. `대여가능`, `15+일 대여`를 위한 local 필드/검색 필터/관리 API 범위를 잠근다.
5. 가격정책 정리 phase와 차량운영관리 phase를 나눠 승인받는다.
