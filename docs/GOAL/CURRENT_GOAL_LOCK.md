# CURRENT GOAL LOCK - 경쟁사 가격 모니터링 MVP

## Primary Goal
- 현재 가장 중요한 목표: **경쟁사 가격 모니터링 엔진 MVP를 만들어 운영자가 관제패널에서 경쟁사 가격을 확인하고, 자사 보유 차량의 1주일 할인 레이어 가격을 제안받을 수 있게 한다.**

## Why This Matters
- 목적은 기준가격 전체를 다시 만드는 것이 아니라, **공차를 줄이기 위한 단기 가격 판단**을 빠르게 하는 것이다.
- 경쟁사 가격을 수집·비교해 운영자가 월요일마다 이번 주 평일 미예약 차량에 적용할 할인 판단을 할 수 있어야 한다.

## Current Context
- 큰 축 1: 기존 찜카·IMS·홈페이지 자료를 정리해 수정 가능한 관제 구조로 만든다.
- 큰 축 2: 카카오T, 카모아 등 주요 렌터카 플랫폼의 차량 가격을 모니터링할 수 있는 경쟁사 가격 엔진을 만든다.
- 1차 MVP 범위는 경쟁사 가격 모니터링 엔진이다.
- MVP에서 확인할 가격:
  - 보유 모델별 경쟁사 1일 가격
  - 보유 모델별 경쟁사 3일 가격
  - 보유 모델별 경쟁사 5일 가격
  - 지역별 1일 가격
- 초기 MVP는 Vercel/API/운영 DB에 올리지 않고 내부 실행 엔진으로 둔다.
- 초기 결과물은 HTML 리포트, CSV, JSON 파일로 확인한다.
- 보유 모델 기준은 고정 테이블을 먼저 만들기보다, 필요 시 홈페이지 검색 결과 기준으로 조회·매칭한다.
- 예약일은 오늘/내일/2일 후 등이 모두 가격이 다를 수 있으므로, 수집 기준일과 대여 시작일을 분리해 기록해야 한다.
- 가격변동 운영방침:
  - 기준 가격 자체를 바꾸지 않는다.
  - **1주일짜리 할인 레이어**를 쌓아서 할인한다.
  - 대상은 이번 주 예약이 없는 차량과 평일 중심이다.
  - 운영자는 월요일에 한 번 조정하는 흐름을 기본으로 본다.
- 자동 반영과 기준가격 직접 수정은 MVP 범위에서 제외한다.

## Success Criteria
- 관제패널에서 경쟁사별/모델별/지역별 가격을 확인할 수 있다.
- 1일/3일/5일 대여 조건별 가격 비교가 가능하다.
- 조회 기준일, 대여 시작일, 대여 기간, 지역, 플랫폼, 모델 매칭 상태가 함께 남는다.
- 자사 가격 대비 경쟁사 가격 차이를 보고 적절한 할인 제안을 계산한다.
- 제안은 자동 적용하지 않고, 운영자가 검토할 수 있는 상태로만 제공한다.
- 기준가격 수정 없이 1주일 할인 레이어 설계로 이어질 수 있다.

## Active Constraints
- 목표 문서는 실행 승인 문서가 아니다.
- 코드 수정, DB migration, 배포, 외부 API 호출, 크롤링 실행, 운영 설정 변경은 별도 승인 전 금지한다.
- 초기 MVP에서는 운영 DB/Supabase migration/Vercel 배포/관리자 API 변경을 하지 않는다.
- `.env`, token, credential, Vercel env, Supabase 운영 DB, cron/launchd 설정은 protected target이다.
- 경쟁사 서비스 접근은 약관, 차단, 법적 리스크, 호출량 제한을 확인한 뒤 설계한다.
- 자동 가격 반영은 MVP에서 제외한다.
- 기준가격 직접 수정은 MVP에서 제외한다.
- 가격 제안은 참고값이며 운영자의 최종 확인 전 실제 반영하지 않는다.

## Next Actions
1. 현재 홈페이지 검색/가격 계산/프라이싱 허브 구조를 다시 확인한다.
2. 내부 실행 엔진의 입력 조건을 잠근다: 플랫폼, 지역, 대여 시작일, 기간, 모델 매칭 기준.
3. 파일 기반 수집 데이터 구조를 설계한다: raw snapshot, normalized price, model match, 지역, 기간, 수집시각.
4. 내부 리포트 MVP 화면 정보를 설계한다: 모델별 1/3/5일, 지역별 1일, 자사 가격 대비 차이, 추천 할인.
5. 할인 레이어 정책을 설계한다: 1주일 유효기간, 평일 대상, 미예약 차량 대상, 자동반영 제외.
6. Phase 문서에서 구현 단계를 나누고, 각 단계별 검증 방법과 중단 조건을 잠근다.
