# CURRENT GOAL LOCK - 홈페이지 전체 리뉴얼

## Primary Goal
- 현재 가장 중요한 목표: **새로 검증한 `/color-preview` 스타일을 기준으로, 기존 기능과 동작을 유지한 채 전체 홈페이지를 일관된 리뉴얼 스타일로 재탄생시킨다.**

## Why This Matters
- 현재 `/color-preview`에서 모바일 중심 검색 플로우, 카드형 UI, 펄스 안내 UX, 새 톤앤매너가 어느 정도 검증되었다.
- 하지만 실제 서비스 전체는 홈, 검색 결과, 상세, 예약, 회원, 관리자 화면의 스타일과 사용 흐름이 아직 섞여 있다.
- 새 디자인만 입히는 것이 아니라 기존 예약/검색/결제/회원 기능이 깨지지 않아야 운영 가능한 홈페이지 리뉴얼이 된다.

## Current Context
- 기준 스타일 화면: `/color-preview`
- 최근 완료된 작업:
  - `/color-preview` 예약 가능 차량 검색 플로우 정리
  - 딜리버리 위치 → 날짜/시간 → 운전자 연령 → 검색 연결
  - 전체 플로우 펄스 UX 적용 확인
  - 반납일 검색 한도 오늘 기준 30일로 통일
  - production 배포 완료
- 리뉴얼 전략:
  - 전체를 새로 따로 만들기보다, 기존 페이지를 하나씩 교체/정리한다.
  - 단, 복잡한 화면은 임시 컴포넌트로 먼저 만든 뒤 교체할 수 있다.

## Success Criteria
- `/color-preview` 스타일 기준이 홈(`/`)과 주요 사용자 플로우에 일관되게 적용된다.
- 기존 동작이 유지된다.
  - 딜리버리 위치 선택
  - 날짜/시간 선택
  - 운전자 연령 선택
  - 차량 검색
  - 차량 상세 진입
  - 예약 진행
  - 회원/비회원 예약조회
  - 관리자 핵심 기능
- 모바일과 PC 모두 사용 가능한 레이아웃으로 정리된다.
- 각 phase마다 빌드 또는 화면 확인 등 최소 검증 근거를 남긴다.
- 완료된 phase는 문서화하고 필요한 경우 커밋한다.

## Active Constraints
- 목표 문서는 실행 승인 문서가 아니다.
- 코드 수정, DB 변경, 배포, 외부 서비스 반영은 별도 승인 범위 안에서만 진행한다.
- 기능을 새로 만들기보다 기존 검증된 동작을 보존하는 것을 우선한다.
- 예약/결제/인증/관리자 기능은 디자인 변경 중 깨지면 즉시 중단하고 원인 보고 후 재승인받는다.
- `.env`, secret, runtime config, DB migration, 결제/인증 설정은 protected target이다.
- `/color-preview` 스타일을 기준으로 삼되, 각 페이지의 기능 목적에 맞게 과도한 장식은 줄인다.
