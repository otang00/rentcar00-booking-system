# ADMIN BOOKING CHANGE UI PHASE PLAN

## Purpose
관리자 예약확인 화면 안에서 날짜/차량 변경을 처리하는 내부 운영용 UI를 확정한다.

## Confirmed UI Direction
- 고객 화면에는 예약 변경 기능을 노출하지 않는다.
- 관리자 예약확인 화면 하단에 `예약 변경` 버튼을 둔다.
- 버튼 클릭 시 같은 화면 안에 접이식 변경 패널을 연다.
- 변경 유형은 세 가지로 고정한다.
  - 날짜 변경
  - 차량 변경
  - 날짜+차량 변경
- 날짜 변경은 기존 차량을 유지한다.
- 차량 변경/날짜+차량 변경은 차량 ID 직접 입력이 아니라 관리자 차량 검색 후 결과 선택으로 진행한다.

## Change Panel Fields
- 새 대여일시
- 새 반납일시
- 새 차량 검색/선택
  - 차량번호
  - 차량명
  - source car id
  - 예약 겹침 여부
- 변경 사유
- 저장 전 확인
  - 현재 일정
  - 변경 일정
  - 현재 차량
  - 변경 차량
  - 결제/환불 자동처리 없음 안내

## Data Display Rule
- 고객 예약조회는 `booking_orders` 현재값을 읽는다.
- 변경 저장 후 고객에게는 변경된 날짜/차량이 표시된다.
- 원장에 변경 전/변경 후 필드를 따로 만들지 않는다.
- 변경 전/후 추적은 이벤트 로그 payload에 남긴다.

## Amount Rule
- 1차 UI에서는 결제/환불/차액 정산을 자동 처리하지 않는다.
- 금액 저장 정책은 API phase에서 별도 확정한다.
- 차액은 필요 시 참고값으로만 다루고 별도 차액 필드는 만들지 않는다.

## Exclusions
- 고객용 변경 신청 UI
- 자동 결제
- 자동 환불
- 별도 차액 필드 추가
- 시작된 예약 강제 변경 UI

## Verification
- `npm run build`
- 관리자 변경 UI가 기존 예약확인 화면 안에서 열리고 닫히는지 확인
- 차량 변경 시 검색 결과에서 차량을 선택해야 저장 가능하도록 제한
