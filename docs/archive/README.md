# ARCHIVE

현재 기준에서 직접 사용하지 않는 문서, 폐기된 계획, stale baseline, 과거 참고 자료를 둔다.

## 원칙
- 삭제보다 archive 이동을 기본으로 한다.
- 왜 archive인지, 현재 기준으로 쓰면 안 되는 이유, 대체 기준이 있으면 함께 기록한다.
- 새 문서는 `GOAL / PHASE / COMPLETED / ARCHIVE` 4축 기준을 우선한다.

## 참고
기존 프로젝트에는 과거 구조인 `docs/archive`, `docs/past`, `docs/complete`, `docs/present`가 남아 있다. 대량 이동은 별도 승인과 충돌 검토 후 진행한다.
