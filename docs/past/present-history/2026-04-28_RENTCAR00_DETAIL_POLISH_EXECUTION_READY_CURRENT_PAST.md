## 0. 목적

이번 작업은 기능 추가가 아니라 **상세페이지 사용감 마감**이다.

핵심 목표는 아래 5개다.

1. 딜리버리 주소 선택 모달의 스크롤/정렬 문제 해결
2. 로그인 회원 운전자 정보 잠금 상태를 시안적으로 더 명확하게 표시
3. `수정` 확인 모달의 줄바꿈/밀도 개선
4. 상세 하단 footer 정보 영역을 기존 사이트 톤을 참고해 더 읽기 좋게 정리
5. 인증문자 안내 문구에서 `3분 내로 입력` 의미 문구 제거

이번 문서는 **실행 전 범위 고정**이 목적이다.
코드 수정은 이 문서를 기준으로 phase 단위로 진행한다.

## 1. 이번에 고정할 문제 정의

### 1-1. 딜리버리 주소 모달
현재 확인된 문제
- 2단계/3단계 리스트가 선택 상태여도 맨 위 정렬이 아니라 중간 애매한 위치에서 보일 수 있다.
- 선택 후 다시 열었을 때 선택 위치가 보장되지 않는다.
- 단계 이동 후 스크롤 위치가 사용자의 기대와 다르게 남는다.

이번에 잠글 방향
- 선택된 시/구/동 항목은 모달 진입 시 **눈에 바로 보이는 위치**로 정렬한다.
- 가능하면 `scrollIntoView({ block: 'start' })` 성격으로 맞춘다.
- 3단계는 선택 지역 카드를 **상단 기준 위치**로 맞추는 쪽을 우선한다.
- 단계 전환 시 이전 단계의 중간 스크롤 잔상은 남기지 않는다.
- 모바일/데스크톱 모두 같은 원칙을 쓴다.

구현 우선순위
1. selected province/city/dong ref 기반 스크롤 보정
2. reopen 시 selected path 복원 후 자동 정렬
3. 필요 시 3단계 상단에 현재 선택 요약 카드 고정

### 1-2. 로그인 회원 운전자 정보 잠금 상태
현재 확인된 문제
- 로그인 상태에서 상세 진입 시 실제로는 잠금 상태인데, 시각적으로는 일반 입력폼과 크게 다르지 않다.
- 사용자는 왜 입력이 안 되는지 한 번에 이해하기 어렵다.

이번에 잠글 방향
- 잠금 상태에서는 일반 disabled 느낌보다 **읽기 전용 요약 카드 + 잠금 폼**에 가깝게 보이게 한다.
- 일반적으로 많이 쓰는 방식으로 아래를 반영한다.
  - 입력 배경을 살짝 진하게/회색 톤으로 조정
  - 테두리/텍스트 대비를 읽기 전용 상태답게 정리
  - 상단에 `회원 정보로 잠금됨` 성격의 상태 배지 또는 안내문 추가
  - `수정` 버튼은 그대로 유지
- 잠금 해제 후 수정 모드에 들어가면 다시 일반 입력폼 대비가 살아나야 한다.

### 1-3. 운전자 정보 수정 확인 모달
현재 확인된 문제
- 확인 모달 문장 줄바꿈이 좁은 폭에서 어색하다.
- 경고/안내/행동 버튼 구분이 약하다.

이번에 잠글 방향
- 모달 폭, 문장 길이, 버튼 구성을 조금 더 읽기 쉽게 정리한다.
- 아래 구조를 기본으로 한다.
  - 제목 1줄
  - 설명 2~3줄
  - 영향 요약 1개 박스 또는 bullet
  - 하단 행동 버튼
- 핵심 경고는 `입력값 초기화 후 재인증 필요`가 한 번에 읽혀야 한다.

### 1-4. footer 정보 영역
현재 확인된 문제
- 현재 footer는 법인명/대표/사업자/주소/전화 수준이라 정보 구조가 단조롭다.
- 사용자가 운영시간, 연락 채널, 방문 정보 같은 실제 문의용 정보를 하단에서 바로 읽기 어렵다.

참고 기준
- 참고 사이트: `https://rentcar00.cafe24.com` (2026-04-28 확인)
- 하단 참조 요소:
  - 전화 문의
  - 카카오톡 ID
  - 방문 주소
  - 운영시간
  - 점심시간

이번에 잠글 방향
- footer는 법적 정보 + 실제 문의 정보가 분리되어 보여야 한다.
- 단, **법인 정보는 유지**하고, 그 아래/옆에 문의 요약을 붙인다.
- 과도한 복제는 하지 않고 현재 앱 톤에 맞게 정리한다.
- 클릭 가능한 행동 요소를 검토한다.
  - 전화
  - 카카오 채널/ID
  - 지도 링크

중요 확인 포인트
- 현재 로컬 회사 정보에는 `02-592-0079`가 있고,
- landing/cafe24 쪽에는 `010-2416-7114`가 노출된다.
- 즉 **전화번호 소스가 불일치**한다.
- footer 개편 시 전화번호 소스는 임의 통일하지 말고, 현재 정책/실사용 기준을 먼저 잠가야 한다.

### 1-5. 인증문자 안내 문구
현재 추가 요청
- 인증문자 본문 또는 인증 안내 문구에 들어간 `3분 내로 입력` 의미 문구는 제거한다.

이번에 잠글 방향
- 인증 유효시간 로직 자체는 바꾸지 않는다.
- 다만 사용자에게 보이는 SMS/안내 문구에서는 `3분 내 입력` 표현을 제거한다.
- 남길 경우에도 단순 `인증번호` 전달 중심 문구로 정리한다.
- 이 항목은 실제 발송 문구와 화면 보조문구를 함께 점검 대상으로 본다.

## 4. Phase 계획

### Phase 1. 딜리버리 주소 모달 정렬/스크롤 고정
목적
- reopen/step-change/select 이후에도 현재 선택 위치가 즉시 보이게 만든다.

수정 범위
- `DeliveryLocationModal.jsx`
- 필요 시 `styles.css`

종료 조건
- desktop: province/city/dong selected 항목이 reopen 시 보인다.
- desktop: selected dong 카드가 상단 기준에 가깝게 정렬된다.
- mobile: step 변경 시 이전 스크롤 잔상이 남지 않는다.
- 선택 후 다시 열어도 path 복원과 시야 정렬이 같이 된다.

### Phase 2. 로그인 회원 잠금 상태 시각화
목적
- 잠금 상태가 읽기 전용 상태로 한눈에 보이게 만든다.

수정 범위
- `CarDetailSection.jsx`
- `styles.css`

종료 조건
- 회원 진입 시 잠금 상태 배경/테두리/안내가 일반 입력 상태와 구분된다.
- `수정` 버튼의 의미가 더 분명해진다.
- 수정 모드 전환 시 편집 상태와 잠금 상태 대비가 명확하다.

### Phase 3. 수정 확인 모달 가독성 개선
목적
- 줄바꿈, 정보 밀도, 경고 메시지 구조를 읽기 좋게 정리한다.

수정 범위
- `CarDetailSection.jsx`
- `styles.css`

종료 조건
- 좁은 화면에서도 제목/설명/버튼이 어색하게 꺾이지 않는다.
- `초기화 후 재인증 필요` 메시지가 한 번에 이해된다.

### Phase 4. footer 정보 재정리
목적
- 법적 정보 + 실사용 연락 정보를 구조적으로 분리한다.

수정 범위
- `Layout.jsx`
- `styles.css`
- 필요 시 `src/data/mock.js` 또는 `src/data/landing.js` 참조 구조 정리

종료 조건
- footer에서 법인 정보와 문의 정보가 구분된다.
- 운영시간/카카오/방문 정보가 읽기 쉬워진다.
- 전화번호 기준은 임의로 바꾸지 않고 잠근 소스를 따른다.

### Phase 5. 인증문자/안내 copy 정리
목적
- 사용자에게 보이는 인증 관련 문구에서 `3분 내 입력` 표현을 제거한다.

수정 범위
- OTP/SMS 템플릿 관련 서버 파일
- OTP 보조 문구가 있는 프론트 파일

종료 조건
- 사용자 노출 인증 문구에서 `3분 내로 입력` 의미 문구가 제거된다.
- 인증 TTL 로직은 유지된다.

### Phase 6. 검증
목적
- 데스크톱/모바일에서 핵심 흐름이 깨지지 않는지 확인한다.

검증 항목

- 상세페이지 로그인 상태 진입
- 수정 모달 열기

## 5. 실행 잠금 규칙

- 이번 phase는 **상세페이지 마감 다듬기**에만 집중한다.
- DB, API 계약, 인증 로직 의미 변경은 넣지 않는다.

- 보험/유의사항 전문 개편은 이번 phase 주범위가 아니다.
- 수정은 반드시 phase 단위로 진행하고, 각 phase마다 build 또는 확인 기준을 둔다.
