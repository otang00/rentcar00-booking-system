# RENTCAR00 PRICING_HUB UI rules current

Last updated: 2026-04-29

이 문서는 관리자 UI의 active 표시/편집 규칙을 잠근다.

---

## 1. 화면 목표

관리자는 아래만 하면 된다.

1. 그룹 선택
2. 기존 기준금액 확인
3. 기준 24시간 금액 조정
4. 주중 % / 주말 % 조정
5. 실시간 계산값 확인
6. `수정` 버튼으로 저장

---

## 2. 보여줄 것

### 그룹 목록
- 그룹명
- 정책명
- 현재 주중24
- 현재 주말24

### 선택 그룹 카드
- IMS 그룹 ID
- 그룹명
- 연결 정책명
- 해당 그룹 차량번호 목록

### 편집 카드
- `수정` 버튼 1개
- `불러온 최초 기준금액`
- `기준 24시간 금액`
- `주중 24시간 요금`
- `주말 24시간 요금`

---

## 3. 편집 가능 인자

편집 가능한 값은 아래 3개만 둔다.

1. 기준 24시간 금액
2. 주중 %
3. 주말 %

나머지 값은 직접 편집하지 않는다.

---

## 4. 배치 규칙

### 기준 24시간 금액
- 한 줄 유지
- 오른쪽 조작부 배치
- input 양옆에 `- / +` 버튼 배치
- 세로 스피너형 조작은 쓰지 않음

### 주중 % / 주말 %
- 각 줄은 한 줄형으로 유지
- `- [숫자입력] +` 형태
- `%`는 input 내부 오른쪽에 흐리게 표시
- 숫자 input 은 작고 압축된 폭 유지

### 표시 최소화
- 하단 중복 비율 요약은 두지 않음
- 복잡한 구조 설명 텍스트는 넣지 않음
- 사용자가 필요한 실금액과 조작부만 남김

---

## 5. 숨길 것

- legacy JSON 카드
- 범용 rate_scope 직접 편집
- override 편집
- 내부 metadata
- 중복 계산 설명
- 별도 모드 토글 버튼
  - `기준값수정`
  - `주중요금수정`

---

## 6. 동작 규칙

- 입력값 변경 시 금액은 실시간 반영
- `수정` 버튼은 계산 버튼이 아니라 저장 버튼
- 저장 전 화면 계산값과 저장 후 재조회 값은 같은 규칙을 사용해야 함

---

## 7. 한 줄 결론

**UI는 하나의 카드에서 기준24 / 주중% / 주말%만 짧게 조정하고, 나머지는 실시간 계산값으로 보여주는 구조로 잠근다.**
