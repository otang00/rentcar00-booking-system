# DECISIONS

## 2026-04-15

### 1. 검색은 DB-only 를 유지한다
- `/api/search-cars` 는 partner 검색으로 되돌리지 않는다.
- 이후 검색 개선은 모두 DB-only 전제에서 진행한다.

### 2. fix-forward 로 간다
- 검색 품질 문제는 현재 DB 경로 위에서 순차 보정한다.
- 치명적 장애가 아닌 이상 partner 복구를 기본 대응으로 삼지 않는다.

### 3. 우선순위 1, 2는 delivery 정책 DB화로 처리 완료했다
- `delivery_regions` 를 검색의 delivery 정책 source of truth 로 사용한다.
- partner 에서 확인한 허용 동/금액 리스트를 DB 정책으로 옮겼다.
- `deliveryPrice` 는 지역별 `round_trip_price` 를 그대로 사용한다.

### 4. 21세 결과 0건은 현재 DB 데이터 기준 정상으로 본다
- 활성 차량 전체의 `rent_age` 최솟값이 26이다.
- 따라서 `driverAge=21` 검색 0건은 현재 로직 오류보다 데이터 상태에 가깝다.

### 5. 현재 남은 우선순위는 2개로 잠근다
1. 가격표/운영 설정 보강
2. 검색-상세 불일치 점검

### 6. 우선순위는 순서대로 처리한다
- 한 번에 여러 축을 같이 건드리지 않는다.
- 각 우선순위는 재현 -> 수정/DB추가 -> 검증 순서로 끝낸다.

### 7. 코드 버그와 데이터 부족을 분리한다
- 정렬 누락, 필터 누락, 매핑 오류는 코드 문제다.
- 요금표 부족, delivery 정책 부족, 운영 설정 부족은 데이터 문제다.

### 8. 검색 안정화 전에는 상세/예약으로 확장하지 않는다
- `/api/car-detail` 및 예약 플로우는 다음 단계다.
- 지금은 검색 품질 안정화가 최우선이다.
