## 1. 남은 작업 목록
### A. 보호형 상세 경로 전환
목표
- `/admin/booking-confirm?token=...` 중심 구조를 줄이고
- `/admin/bookings/:id` 같은 보호형 상세 경로로 정리한다.

남은 이유
- 현재는 권한은 잠겼지만 URL 구조는 여전히 token 기반이다.
- 장기적으로는 관리자 시스템 경로와 권한 모델을 일치시키는 편이 낫다.

완료 조건
- 관리자 상세 기본 진입점이 token 링크가 아니라 보호형 경로가 된다.

---

### B. 관리자 목록 즉시 취소 액션 여부 결정
목표
- 관리자 목록에서 상세 진입 없이 바로 취소할 수 있게 할지 결정한다.

남은 이유
- 현재는 상세에서만 취소 가능하다.
- 운영 속도는 느리지만 오조작 위험은 낮다.

완료 조건
- 유지 또는 추가 중 하나로 정책 확정
- 추가 시 권한/확인 모달/문구까지 같이 잠금

---

### C. 관리자 판별 방식 개선
목표
- 프론트/서버의 관리자 판별을 이메일 화이트리스트 의존에서 role 기반으로 이동할지 검토한다.

남은 이유
- 현재는 빠른 운영 기준으로 충분하지만 확장성은 낮다.

완료 조건
- 유지 또는 role 기반 전환 여부 확정
- 전환 시 profiles/claims/source of truth 까지 잠금

---

### D. 관리자 취소 후속 운영 정리
목표
- `admin_cancelled_after_start` 이후 운영 화면/내부 처리 기준을 더 명확히 한다.

남은 이유
- 이벤트는 분리했지만 정산/차량상태/후속 운영 표시는 아직 최소 수준이다.

완료 조건
- 필요 시 관리자 화면 노출 문구 또는 후속 상태 처리 기준 정리
