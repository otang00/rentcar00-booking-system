## 1. 목적
전체 페이지 상단 헤더를 하나의 기준으로 통일하고, 모바일에서 글자가 세로로 밀리거나 레이아웃이 무너지는 문제를 구조적으로 해결한다.

## 2. 현재 상태
1. 헤더가 2종으로 분리되어 있다.
   - `Layout.jsx` 기반 일반 헤더
   - `BrandHeader.jsx` 기반 랜딩 헤더
2. 일반 헤더는 `header-three-col` 3열 구조다.
3. 랜딩 헤더는 별도 `landing-header-*` 스타일 체계를 사용한다.
4. 관리자 메뉴 추가 후 일반 헤더의 우측 폭 부담이 커졌다.
5. 모바일 기준 일관된 축약/숨김/메뉴화 규칙이 없다.

## 3. 현재 문제 정의
1. 모바일에서 텍스트가 세로로 밀리거나 비정상 줄바꿈이 발생할 수 있다.
2. 페이지별 헤더 구조가 달라 전체 제품 인상이 통일되지 않는다.
3. 일반 헤더는 고정 3열 + 고정 높이 조합이라 작은 화면에서 취약하다.
4. 회원명/관리자 메뉴/로그인 링크가 누적되면 우측 메뉴가 쉽게 붕괴한다.

## 4. 핵심 원인
1. `grid-template-columns: 240px minmax(0,1fr) 240px`
2. `height: 96px` 고정
3. 중앙 타이틀의 모바일 우선순위가 낮은데도 계속 공간을 차지함
4. 메뉴 액션 증가 대비 모바일 전략 부재
5. 헤더 시스템 이원화로 CSS 보정이 흩어져 있음

## 5. 권장 방향
1. 헤더는 장기적으로 `공통 헤더 1종` 으로 통합한다.
2. 랜딩/일반 차이는 variant 로만 처리한다.
3. 모바일에서는 중앙 타이틀을 제거 또는 축약한다.
4. 모바일 우측 액션은 줄바꿈보다 `메뉴화` 를 우선한다.
5. 관리자 메뉴/사용자명은 모바일에서 축약 또는 숨김 처리한다.

## 6. 공통 헤더 목표안
### 공통 컴포넌트
- 가칭: `AppHeader`

### variant
- `default`
- `landing`

### 상태별 노출 기준
- 비로그인: 로그인 / 회원가입 / 필요시 비회원 예약조회
- 로그인 회원: 예약내역 / 로그아웃
- 관리자: 예약관리 추가
- 모바일: 핵심 1~2개 액션 우선, 나머지 메뉴화

## 7. 모바일 기준 잠금
1. 360 / 390 / 430 폭 기준 검증
2. 타이틀은 축약 또는 제거 가능
3. 텍스트는 `nowrap + ellipsis` 우선
4. 헤더 높이는 `min-height` 기반
5. 우측 액션은 풀 노출보다 접힘 구조 우선

## 11. 구현 전 검증 포인트
- 헤더를 하나의 시스템으로 설명 가능한가
- 모바일 텍스트 세로 밀림 가능성이 구조적으로 제거되는가
- 관리자 메뉴가 추가돼도 헤더가 다시 깨지지 않는가
- 랜딩/일반/관리 페이지가 동일한 제품 톤으로 보이는가

## 12. 현재 권장 결론
- 지금은 부분 CSS 땜질보다 헤더 시스템 통합이 맞다.
- 먼저 구조와 모바일 기준을 잠그고 들어가야 재작업이 줄어든다.
- 실행 준비는 완료 가능하며, 구현은 다음 승인 후 phase 단위로 진행한다.
