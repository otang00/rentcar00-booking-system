# GOAL

## 현재 최우선 목표
`rentcar00-booking-system`을 실제 운영 가능한 렌터카 홈페이지/예약 플랫폼으로 안정화한다.

## 기준
- 기존 검색, 상세, 예약, 결제, 회원, 관리자 기능을 깨지 않는다.
- 홈페이지 원장은 `booking_orders` 기준이며, 실제 차량 운영 원장은 IMS가 최종 기준이다.
- 외부 연동(KCP, 카모아, 찜카, OPS 앱)은 코드보다 상태 owner와 이벤트 흐름을 먼저 확인한다.
- 가격 truth는 홈페이지 DB/가격 허브와 외부 플랫폼 반영값이 어긋나지 않도록 관리한다.

## 완료 기준
- 고객 플로우: 검색 → 상세 → 예약정보 → 결제 → 예약확정 → 조회가 검증된다.
- 관리자 플로우: 예약 확인/변경, 가격 확인/수정, 동기화 상태 확인이 가능하다.
- KCP 운영 결제 승인 가능 상태가 확인된다.
- 카모아/찜카 등 외부 플랫폼 반영은 실행 결과와 복구 근거가 남는다.
- 주요 상태와 이벤트 흐름이 문서화되어 다음 작업이 구조를 더 꼬이게 만들지 않는다.

## 하지 않을 것 / 승인 필요
- 승인 없는 코드 수정, DB 변경, 배포, 런타임 설정 변경, 외부 서비스 저장은 하지 않는다.
- `.env*`, secret, token, 인증서, launchd, cron, Vercel env는 별도 승인 없이는 수정하지 않는다.
- 가격/예약/결제 상태를 편의상 직접 바꾸지 않는다.
